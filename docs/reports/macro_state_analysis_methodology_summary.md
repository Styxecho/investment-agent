# 宏观状态诊断方法论

## 一、核心设计理念
- **目标**：通过三维同构框架（增长 × 通胀 × 流动性）判定当前宏观状态，供投资研究与资产配置参考。
- **输出**：携带动量特征的定性状态标签（如“扩张上行_温和上行_双宽”），以及供策略层直接调用的宏观象限标签（如“强势复苏”），非直接买卖信号。
- **哲学**：
  - **水平与方向解耦**：水平衡量经济所处的绝对位置（扩张/收缩、高/温/低通胀等），方向衡量经济边际上的动态走向（上行/下行/平稳）。两者独立计算，最终合成。
  - **逻辑完全同构**：增长、通胀、流动性三大维度均严格遵循相同的底层架构。
  - **信息保真原则**：在底层向量拼接时，绝不将“水平+方向”降维折叠，必须保留完整的向量信息；仅在最终输出给策略层时，才通过规则引擎映射为宏观象限。

## 二、宏观底层数据源明细
为保证因子构建的严谨性与可落地性，各维度所需获取的基础数据源及频率要求如下：

### 2.1 增长维度数据源
- **PMI（采购经理指数）**：
  - **指标**：中国制造业PMI。
  - **频率/来源**：月度 / 国家统计局。
  - **特性**：扩散指数，天然具备环比含义，直接取原始值对标50荣枯线，**不参与HP滤波**。
- **非制造业PMI（服务业PMI）**：
  - **指标**：非制造业商务活动指数。
  - **频率/来源**：月度 / 国家统计局。
  - **特性**：用于对冲纯工业指标的盲区，触发结构性仲裁（原始值对标50）。
- **IAV（工业增加值）**：
  - **指标**：规模以上工业增加值**当月同比增速**。
  - **频率/来源**：月度 / 国家统计局。
  - **特性**：直接获取同比数据，进入统一滤波流水线提取动量。

### 2.2 通胀维度数据源
- **核心CPI（核心居民消费价格指数）**：
  - **指标**：核心CPI**同比增速**（剔除食品和能源）。
  - **频率/来源**：月度 / 国家统计局（需自行剥离计算或使用第三方口径）。
  - **特性**：一方面直接用原始同比值判定通胀“水平”；另一方面，对其同比序列做滤波提取“方向”。
- **PPI（工业生产者出厂价格指数）**：
  - **指标**：PPI全部工业品**同比增速**。
  - **频率/来源**：月度 / 国家统计局。
  - **特性**：直接获取同比数据，进入统一滤波流水线提取动量。

### 2.3 流动性维度数据源
- **M2（广义货币供应量）**：
  - **指标**：M2**存量同比增速**。
  - **频率/来源**：月度 / 中国人民银行。
  - **特性**：直接获取同比数据，进入统一滤波流水线提取动量（提取的周期项即广义“货币脉冲”）。
- **社融（社会融资规模）**：
  - **指标**：社会融资规模**存量同比增速**。
  - **频率/来源**：月度 / 中国人民银行。
  - **特性**：直接获取同比数据，进入统一滤波流水线提取动量（提取的周期项即“信用脉冲”）。
  - **【注：财政干扰风险】**社融数据受政府债券集中发行干扰极大。当巨额特别国债或地方专项债集中缴款时，实体企业未拿到钱，资金沉淀在国库，会导致社融存量同比骤升，形成“信用扩张”的假象。**系统底层必须记录此风险，当检测到社融同比或其滤波周期项出现不符合实体信贷规律的极端单月跳变时，触发人工核查预警。**
- **DR007（存款类金融机构7天期回购利率）**：
  - **指标**：DR007日频加权利率。
  - **频率/来源**：日频 / CFETS。
  - **特性**：作为市场利率代表，用于识别利率趋势拐点和央行政策传导效果。日频数据参与拐点识别，月度加权均值仅作参考。
- **政策利率（7天逆回购利率）**：
  - **指标**：央行公开市场7天期逆回购操作利率。
  - **频率/来源**：日频（仅在操作日有值）/ 中国人民银行。
  - **特性**：作为DR007的基准锚和央行意图的直接信号。非操作日采用前向填充（上一次操作利率持续有效至下次操作）。日频数据用于识别央行政策拐点，月末有效利率用于水平参考。

## 三、全局数据预处理规范（核心流水线）
此章节为除PMI和非制造业PMI之外，**所有指标（IAV、核心CPI、PPI、M2、社融）**提取“方向与动量”的统一强制性流水线。

### 3.1 核心数学逻辑：用“二阶导”吸收基数与春节效应
- **绝对禁令**：**严禁尝试对这些指标进行复杂的“环比转换 -> 春节回归 -> X-13季调”**。一方面环比绝对值获取困难且累积误差大，另一方面季调参数的主观设定容易引入新噪音。
- **降维打击**：直接对**原始同比增速序列**进行单向HP滤波。在数学上，对增速序列提取周期项，等价于提取宏观变量的“加速度（二阶导）”。这种方法能**自动且完美地吸收掉绝大部分的基数效应错位以及春节错月导致的同比毛刺**，实现极简与鲁棒的统一。

### 3.2 统一动量提取步骤
1. **输入**：各指标的原始当月同比增速序列（如 IAV当月同比、PPI同比、M2同比等）。
2. **单向HP滤波**：对输入序列使用 **单向HP滤波** 提取周期项。
   - **严禁使用双向HP滤波**（会导致当期值受未来数据影响，产生端点漂移与未来函数）。
   - **参数设定**：月度数据统一采用 $\lambda = 129600$。
3. **标准化**：计算周期项过去36个月滚动Z-score。
4. **自适应偏离度计算**：`deviation = 当前Z-score - MA3(Z-score)`。

### 3.3 自适应偏离阈值计算
- **废弃全局固定阈值**。采用自适应阈值：`Threshold = ± 1.0 × 过去36个月 deviation 序列的滚动标准差`。以此解决不同指标波动率差异导致的信号过敏或迟钝问题。

## 四、增长维度（Growth = PMI + IAV + 非制造业PMI）

### 4.1 水平判定
- **PMI水平**：PMI原始值 ≥ 50 → **扩张**；< 50 → **收缩**。
- **IAV水平**：IAV当月同比经单向HP提取的 **周期值** ≥ 0 → **扩张**；< 0 → **收缩**。
- **合成（情境与结构仲裁）**：
  - 同向时：取同向结果。
  - 分歧时（PMI扩张 + IAV收缩）：若PMI趋势方向=上行 → **扩张**；否则 → **中性**。
  - 分歧时（PMI收缩 + IAV扩张）：若PMI趋势方向=下行 → **收缩**；否则 → **中性**。
  - **结构性防误判仲裁**：若 PMI收缩 + IAV收缩，**但当月非制造业PMI > 53**，则增长水平强制修正为 **中性**。

### 4.2 方向判定（防抖状态机逻辑）
PMI与IAV统一进入状态机，但在前置的Z-score计算上采用不同逻辑，最终殊途同归：
• IAV的Z-score计算：采用第三章标准流水线（滤波周期项 -> 36个月滚动均值与标准差 -> Z-score）。
• PMI的Z-score计算（绝对零点法）：
◦ 核心逻辑：PMI是扩散指数，50代表环比绝对零增长。若使用滚动均值，会因经济长期中枢下移导致偏离度失真。
◦ 计算公式：Z_pmi = (PMI原始值 - 50) / 过去36个月(PMI原始值 - 50)的滚动标准差。
◦ 本质：这样算出的Z-score，等价于“标准化后的环比动能”，与IAV滤波周期项的Z-score在数学含义上完美对齐。
• 进入统一状态机：
1. 计算偏离度：deviation = 当前Z-score - MA3(Z-score)。
2. 基于自适应阈值计算 raw_dir（↑ / ↓ / →）。
3. 趋势方向判定规则：
◦ 首月初始化：raw_dir ≠ → 则 trend_dir = raw_dir；否则 trend_dir = →。
◦ 脱离中性（单次击发）：若当前 trend_dir 为 →，只要当期 raw_dir 出现非中性信号，下个月直接翻转。
◦ 脱离非中性（双重确认）：若当前 trend_dir 为 ↑ 或 ↓，必须 连续两个月 raw_dir 相同才允许翻转。
• 双指标合成：同向取同向；一非中性一中性取非中性；一上一下冲突 → 中性。
### 4.3 状态合成
水平（扩张/中性/收缩） + 方向（上行/平稳/下行） = 9种基础状态。

## 五、通胀维度（Inflation = 核心CPI + PPI）

### 5.1 水平判定
- **核心CPI水平（绝对主锚）**：使用核心CPI**原始同比值**。
  - > 3% → **高通胀**
  - 1% ~ 3% → **温和通胀**
  - < 1% → **低通胀**

### 5.2 方向判定
- **核心CPI方向**：基于“核心CPI同比序列”经统一流水线提取的周期项计算。
- **PPI方向**：基于“PPI同比序列”经统一流水线提取的周期项计算。
- **双指标合成与背离标记**：正常合成逻辑同增长维度。**特殊规则**：若 PPI=↑ 且 核心CPI=↓（或反之），方向判定为 **中性**，但必须打上 **“成本传导背离”** 标签。

### 5.3 状态合成
水平（高/温和/低） + 方向（上行/平稳/下行） = 9种基础状态。

## 六、流动性维度（Liquidity = M2 + 社融 + OMO + DR007）

### 6.1 数量水平判定（沿用V7逻辑，不变）
- **M2水平**：M2存量同比经单向HP提取的 **周期值** ≥ 0 → **货币扩张**；< 0 → **货币收缩**。
- **社融水平**：社融存量同比经单向HP提取的 **周期值** ≥ 0 → **信用扩张**；< 0 → **信用收缩**。
- **合成规则**：以M2水平为主锚。若分歧，取M2水平，社融水平作为辅助修正。

水平判定部分保持稳定，不涉及价格信号。

### 6.2 价格型方向判定（V8核心更新）

方向判定分为三步：先识别央行意图作为主信号，再通过传导验证进行修正，最后与数量方向合成。

#### 6.2.1 央行意图识别（初始方向）

**定义**：依据过去6个月内OMO利率的累计净变化方向判定央行是边际收紧还是边际宽松。

- **净变化** = 当期OMO利率 − 6个月前OMO利率（若期间存在多次调整，则按调整幅度累加）
- **阈值**：累计上调 > 10bp → **收紧**，初始方向 ↓
  　　　　累计下调 > 10bp → **宽松**，初始方向 ↑
  　　　　变化在±10bp以内 → **中性**，初始方向 →

**规则说明**：
- 6个月窗口对应货币政策向实体经济传导的典型时滞。
- 10bp阈值过滤技术性微调（OMO最小调整步长为5bp或10bp）。
- 6个月内若出现先加息后降息，净变化以最终结果为准；若净变化虽小但12个月内曾有显著加息/降息且当前利率处于高位/低位，维持此前的方向判定（以防加息暂停期被误判为中性）。

#### 6.2.2 传导验证（方向修正）

央行意图能否落地，取决于市场利率（DR007）是否跟随政策利率变动，且两者趋势的**先后关系**决定了是主动引导还是被动追赶。

**趋势拐点识别方法**：

**预处理**：
- OMO：取日频政策利率（非操作日前向填充）。DR007：取日频加权利率。
- 分别计算20交易日的简单移动平均（SMA20），以滤除日间噪声。

**突破识别**：
- 计算SMA20的5日差分（diff = SMA20(t) − SMA20(t−5)）。
- 趋势形成：diff连续5个交易日同号（正或负），方向由负转正视为上行突破，由正转负视为下行突破。
- 拐点日期：确认为趋势形成的第一天（即连续同号的第一天）。
- **OMO突变跳过机制**：若OMO单日变动 ≥ 5bp，直接判定为趋势拐点，以变动日为拐点日，不等待SMA确认。

**拐点列表**：对OMO和DR007分别生成历史突破点的日期及方向（上行/下行）。

**先后关系判定**：

取最近一次**同向突破点**进行比较。如两者方向相反，直接判定传导受阻（方向→）。

方向相同时，比较拐点日期：
- 若 OMO拐点日 < DR007拐点日 − 20个交易日（约1个月）：**央行主动引导**
- 若 DR007拐点日 < OMO拐点日 − 20个交易日：**央行被动追赶**
- 否则：视为几乎同步（有效调控）

**若某一方在近12个月内无任何突破点**（趋势水平无方向），则默认另一方为引领者；双方均无趋势则方向→。

**方向修正表**：

| 初始方向（央行意图） | 先后关系 | 修正后价格方向 | 标记 |
|:---|:---|:---|:---|
| 收紧（↓） | OMO主动引导 | ↓ | — |
| 收紧（↓） | 几乎同步 | ↓ | — |
| 收紧（↓） | OMO被动追赶 | ↓ | ⚠️央行被动收紧 |
| 收紧（↓） | 传导受阻（方向相反） | → | — |
| 宽松（↑） | OMO主动引导 | ↑ | — |
| 宽松（↑） | 几乎同步 | ↑ | — |
| 宽松（↑） | OMO被动追赶 | ↑ | ⚠️央行被动宽松 |
| 宽松（↑） | 传导受阻（方向相反） | → | — |
| 中性（→） | DR007上行趋势 | ↑ | ⚠️央行未确认 |
| 中性（→） | DR007下行趋势 | ↓ | ⚠️央行未确认 |
| 中性（→） | DR007无趋势 | → | — |

修正后的价格方向记为 `price_dir`，取值 ↑ / ↓ / →。

[2026-05-12] Note：按此逻辑其实本质就是以央行的意图方向为方向，只有当央行意图和市场现实严重背离的时候才需要进一步判断。故而“领先-滞后”关系的判断其实没有帮助。本版仍然保留此项处理，或许未来这种“领先-滞后”关系会有用处。

#### 6.2.3 数量方向提取（沿用V7逻辑）

M2和社融各自经统一流水线（HP滤波→Z-score→自适应偏离→防抖状态机）产生 `trend_dir`，两者合成规则同增长维度：
- 同向取同向
- 一非中性一中性取非中性
- 一上一下冲突 → 中性

最终得到数量方向 `qty_dir`。

#### 6.2.4 价格方向与数量方向的冲突仲裁

价格方向（短端信号）是央行意图的领先指标，数量方向（长端结果）是滞后验证。两者为上下游关系，非并列。冲突时采用**价格主导、数量验证**的仲裁规则：

| 价格方向 `price_dir` | 数量方向 `qty_dir` | 最终方向 | 标记 |
|:---|:---|:---|:---|
| ↑ | ↑ | ↑ | — |
| ↑ | → | ↑ | — |
| ↑ | ↓ | ↑ | ⚠️数量价格背离 |
| ↓ | ↑ | ↓ | ⚠️数量价格背离 |
| ↓ | ↓ | ↓ | — |
| ↓ | → | ↓ | — |
| → | ↑ | ↑ | ⚠️短端未确认 |
| → | ↓ | ↓ | ⚠️短端未确认 |
| → | → | → | — |

规则要点：
1. 价格方向明确时（↑/↓），最终方向服从价格方向，数量方向仅作为附加验证信息。
2. 当价格方向为中性（→）时，说明央行无边际意图，此时以数量方向的惯性为准，但打上"短端未确认"标记，以降低方向信号的置信度。
3. 价格方向与数量方向相反时，属于深度背离，保留价格方向，并打上"数量价格背离"标记——提示策略层短端与长端在拉锯。

### 6.3 流动性状态合成

组合水平与方向，形成最终标签，示例：

- `双宽↑`：货币信用双扩张，且边际上仍在加速宽松
- `紧货币宽信用↑`（标记：⚠️短端未确认）：信用在扩张，数量方向显示向上，但央行无明确意图，方向可信度下降
- `双紧↓`：货币信用双收缩，且央行主动收紧
- `宽货币紧信用↑`（标记：⚠️数量价格背离）：央行在宽松，但数量信号仍在收缩，可能存在传导时滞或结构性矛盾

流动性状态标签与增长、通胀维度状态标签一同进入宏观象限映射引擎。

### 6.4 流动性状态最终映射表
结合水平与方向，流动性维度最终固化为以下4种基础状态，用于最终宏观映射：
| M2水平状态 \ 社融水平状态 | 信用扩张 | 信用收缩 |
| :--- | :--- | :--- |
| **货币扩张** | **双宽** | 宽货币紧信用 |
| **货币收缩** | 紧货币宽信用 | **双紧** |

*注：方向标签（↑/↓/→及标记）不进入象限映射，仅作为附加信息保留。*

## 七、宏观状态最终映射（工程实施级规则表）

### 7.1 核心原则：分层输出
底层向量拼接保留全信息格式：`[增长状态]_[通胀状态]_[流动性状态]`（如 `扩张上行_温和上行_双宽`）。
在最终输出给策略层或进行归因统计时，将其降维映射为宏观象限标签。此映射采用**优先级规则引擎**模式，按从上到下的顺序进行条件匹配，命中即输出对应象限，不再往下匹配。

### 7.2 宏观象限优先级映射规则表
*注：通配符说明。`扩张*` 代表包含“扩张上行/扩张平稳/扩张下行”；`*下行` 代表任何带“下行”的状态。流动性仅保留6.4节中的4种基础态。*

| 优先级 | 象限标签 | 匹配条件 (增长状态 × 通胀状态 × 流动性状态) | 宏观含义与资产含义 |
| :--- | :--- | :--- | :--- |
| **P1** | **极端滞胀** | `收缩*` 且 `高通胀*` 且 (`双紧` 或 `紧货币宽信用`) | 经济衰退与高通胀并存，且货币收紧。**最危险状态**。股票清仓，做空信用债，持有黄金/现金。 |
| **P2** | **典型滞胀** | (`收缩*` 或 `中性下行`) 且 `高通胀*` 且 非P1流动性 | 经济已落入收缩或快速走向收缩，通胀高企。股债双杀，防御为主（红利/微盘），做窄信用利差。（注：排除了扩张下行，防止过热见顶被误判为滞胀） |
| **P3** | **过热期** | `扩张*` 且 `高通胀*` 且 非P1流动性 | 经济强但通胀越过警戒线。如果流动性紧则是“紧过热”（防范拐点），流动性宽则是“泡沫过热”。逐步兑现权益。 |
| **P4** | **失速衰退** | `收缩*` 且 `低通胀*` 且 (`双紧` 或 `紧货币宽信用`) | 经济失速，通胀极低，但流动性未有效放松（如流动性陷阱）。纯防守，超长利率债有机会。 |
| **P5** | **宽衰退** | (`收缩*` 或 `中性下行`) 且 `低通胀*` 且 (`双宽` 或 `宽货币紧信用`) | 经济弱势，通胀低迷，但流动性已实质宽松。**典型债牛环境**，高股息底仓，成长股左侧试错。 |
| **P6** | **弱复苏** | (`扩张上行` 或 `中性上行`) 且 `低通胀*` 且 (`双宽` 或 `宽货币紧信用`) | 信用未完全起来，但货币先宽，增长刚拐头。通常对应“政策底”到“经济底”过渡。偏防御复苏，利率债和微盘/主题活跃。 |
| **P7** | **强势复苏** | `扩张上行` 且 (`温和通胀*` 或 `低通胀上行`) 且 (`双宽` 或 `宽货币紧信用`) | 经济强劲向上，通胀在舒适区或极低但开始抬头。**最完美股票主升浪环境**，进攻型配置（顺周期/成长）。 |
| **P8** | **完美扩张** | `扩张*` 且 `温和通胀*` 且 非P4流动性 | 经济绝对水平高，通胀温和。若方向为“扩张下行”则需警惕见顶，否则持股待涨。 |
| **P9** | **类衰退过渡** | `中性*` 且 `低通胀*` 且 `双紧` | 处于收缩边缘，流动性被收紧。防范尾部风险，降低仓位。 |
| **P10** | **震荡/观望** | **所有未被上述规则捕获的剩余组合** | 宏观信号混乱（如扩张但通胀低且双紧，或收缩但通胀高且双宽）。系统不发出明确趋势信号，维持现状，依赖其他微观因子。 |

### 7.3 程序实施伪代码示例
开发人员可基于上述规则表，直接写出如下的无歧义匹配逻辑（以Python为例）：
def map_macro_regime(growth_state, inflation_state, liquidity_state):
##Pre-conditions extraction
is_contraction = growth_state.startswith(“收缩”)
is_expansion = growth_state.startswith(“扩张”)
is_exp_up = growth_state == “扩张上行”
is_neutral_down = growth_state == “中性下行”
is_dir_down = growth_state.endswith(“下行”)
is_high_inf = inflation_state.startswith("高通胀")
is_moderate_inf = inflation_state.startswith("温和通胀")
is_low_inf = inflation_state.startswith("低通胀")

##P7 特殊条件：低通胀上行
is_golden_low_inf = (is_low_inf and growth_state.endswith("上行"))

is_tight_liq = liquidity_state in ["双紧", "紧货币宽信用"]
is_loose_liq = liquidity_state in ["双宽", "宽货币紧信用"]

##Priority Matching Engine
if is_contraction and is_high_inf and is_tight_liq:
    return "极端滞胀"
elif (is_contraction or is_neutral_down) and is_high_inf and not is_tight_liq:
    return "典型滞胀"
elif is_expansion and is_high_inf and not is_tight_liq:
    return "过热期"
elif is_contraction and is_low_inf and is_tight_liq:
    return "失速衰退"
elif (is_contraction or is_neutral_down) and is_low_inf and is_loose_liq:
    return "宽衰退"
elif (is_exp_up or growth_state == "中性上行") and is_low_inf and is_loose_liq:
    return "弱复苏"
elif is_exp_up and (is_moderate_inf or is_golden_low_inf) and is_loose_liq:
    return "强势复苏"
elif is_expansion and is_moderate_inf and not is_tight_liq:
    return "完美扩张"
elif growth_state.startswith("中性") and is_low_inf and liquidity_state == "双紧":
    return "类衰退过渡"
else:
    return "震荡/观望"


### 7.4 附录：异常与背离日志拦截（不改变主标签，但触发警告）
除了上述主标签映射，程序在执行 `map_macro_regime` 前，应额外扫描并记录以下四种系统日志，供人工复盘与策略微调：
1. **成本传导背离**：若 PPI方向=↑ 且 核心CPI方向=↓（或反之），无论最终落在哪个象限，系统日志记录 `[WARNING: 成本传导背离，上下游利润可能重塑]`。
2. **结构性假衰退**：若 最终象限包含“衰退”字眼，但当月 `非制造业PMI > 53`，系统日志记录 `[WARNING: 触发服务业对冲，衰退确定性下降]`。
3. **流动性价格-数量背离**：若价格方向与数量方向相反（如 price_dir=↑ 但 qty_dir=↓），系统日志记录 `[WARNING: 流动性价格-数量背离，短端与长端信号拉锯]`。
4. **社融财政干扰嫌疑**：若 社融同比增速或其滤波周期项出现不符合实体信贷规律的极端单月向上跳变（如单月变动超过历史标准差的2倍），系统日志记录 `[WARNING: 社融财政干扰嫌疑，需人工核查是否为政府债集中缴款导致信用扩张失真]`。
