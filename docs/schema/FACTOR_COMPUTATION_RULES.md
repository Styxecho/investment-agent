# 宏观因子计算规则 (Factor Computation Rules)

**版本**: v1.0  
**更新日期**: 2026-04-29  
**适用项目**: Investment Agent - Phase 2.3 宏观信号框架  

---

## 1. 概述

本文档定义所有宏观指标的计算规则，确保因子处理的一致性和可维护性。

### 通用处理流程

```
原始数据 → [特殊预处理] → [频率统一] → HP滤波 → Z-score计算 → Winsorize截断 → 因子值
```

---

## 2. 特殊处理指标

### 2.1 PMI类指标（制造业/非制造业/综合）

| 项目 | 规则 |
|------|------|
| **指标代码** | CN_PMI_MFG_M, CN_PMI_SVC_M, CN_PMI_COMP_M |
| **特殊处理** | 减去基准值 50（荣枯线） |
| **处理公式** | `processed_value = raw_value - 50` |
| **经济含义** | >0: 经济扩张, <0: 经济收缩, =0: 刚好在荣枯线 |
| **处理时机** | 在preprocessor中动态处理（不修改原始数据） |
| **配置参数** | `subtract_baseline: 50` |

**示例**:
- 原始PMI = 51.2 → 处理后 = 1.2（扩张）
- 原始PMI = 49.0 → 处理后 = -1.0（收缩）

---

### 2.2 M1-M2剪刀差

| 项目 | 规则 |
|------|------|
| **指标代码** | CN_M1M2_DIFF_M |
| **特殊处理** | 动态计算 M1_YoY - M2_YoY |
| **处理公式** | `scissor = M1_YoY - M2_YoY` |
| **经济含义** | >0: M1增速>M2，资金活化，利好投资; <0: 资金沉淀，经济偏冷 |
| **处理时机** | 在数据加载阶段从M1和M2原始数据计算 |
| **数据来源** | macro_indicator_value 中 M1_YOY_M 和 M2_YOY_M |

**历史参考值**:
- 2016年底: 剪刀差为负（资金活化不足）
- 2017年初: 快速转正（企业投资意愿增强）
- 2020年: 大幅为正（流动性充裕）
- 2024年: 再度为负（资金沉淀，需求不足）

## 4. 滤波参数配置

### 4.1 统一参数（所有指标）

| 参数 | 值 | 说明 |
|------|-----|------|
| **滤波方法** | one_sided_hp | 单边HP滤波，无未来信息泄露 |
| **λ (lambda)** | 14400 | 月度数据标准值 |
| **预热期** | 18个月 | 前18个月使用双边HP |
| **水平Z-score窗口** | 36个月 | 覆盖一个完整经济周期 |
| **变化率Z-score窗口** | 48个月 | 更长的窗口以稳定标准差估计 |
| **最小样本数** | 12个月 | 计算Z-score的最短数据要求 |
| **Winsorize阈值** | ±3.0 | 截断极端值 |

### 4.2 配置表示例

```json
{
  "indicator_code": "CN_PMI_MFG_M",
  "filter_type": "one_sided_hp",
  "filter_params": "{\"lamb\": 14400, \"subtract_baseline\": 50}",
  "level_window": 36,
  "change_window": 48,
  "winsorize_threshold": 3.0,
  "min_periods_for_zscore": 12,
  "hp_warmup_months": 18
}
```

## 5. 日频数据转月频规则

对于日频指标（南华商品、国债收益率、汇率、DR007、美元指数）：

```
原始日频值 → 20日移动平均 → 取每月最后一个交易日值 → 月频序列
```

**参数**:
- 移动平均窗口: 20日
- 最小有效天数: 10日（避免月初数据不足）
- 取值时点: 每月最后一个交易日

---

## 6. 缺失值处理

| 场景 | 处理方法 | 说明 |
|------|---------|------|
| 单个月缺失 | 线性插值 | 使用前月和后月数据插值 |
| 连续2个月缺失 | 线性插值 | 同上 |
| 超过2个月缺失 | 不插值 | 保持NaN，不参与计算 |
| 开头/结尾缺失 | 删除 | 不参与Z-score计算 |

**特殊案例**:
- 规上工业增加值：1-2月合并发布，2月有值，1月留空，需插值处理

## 7. 因子类型定义

### 7.1 水平因子 (level)

```
level_t = (cycle_t - mean(cycle_{t-window:t-1})) / std(cycle_{t-window:t-1})
```

**含义**: 当前周期项在历史中的相对位置
- >0: 高于历史均值
- <0: 低于历史均值
- |value|>2: 极端偏离

### 7.2 变化率因子 (change)

```
delta_t = cycle_t - cycle_{t-1}
change_t = (delta_t - mean(delta_{t-window:t-1})) / std(delta_{t-window:t-1})
```

**含义**: 周期项变化的加速度在历史中的相对位置
- >0: 变化速度加快（正向加速）
- <0: 变化速度减慢（正向减速或负向加速）
- 用于捕捉拐点信号

## 8. 数据质量检查清单

计算完成后必须验证：

### 8.1 PMI因子
- [ ] 2016年初（经济收缩期）PMI level因子应为负值
- [ ] 2017年（经济扩张期）PMI level因子应为正值
- [ ] PMI cycle值 ≈ 原始值 - 50（经HP滤波后）

### 8.2 M1M2剪刀差
- [ ] 2016年底剪刀差level为负（M2>M1）
- [ ] 2017年初剪刀差快速转正
- [ ] 2020年剪刀差为正（流动性充裕）

### 8.3 通用检查
- [ ] 所有因子值在[-3, 3]范围内（Winsorize生效）
- [ ] 无NaN值（除预热期外）
- [ ] 每个指标有level和change两种因子
- [ ] 日频转月频后数据点数量合理

## 9. 新增指标流程

如需新增宏观指标，按以下步骤：

1. **更新macro_indicators.csv**
   - 添加新行：indicator_code, indicator_name, category, country, frequency, unit

2. **更新macro_indicator_catalog表**
   - 插入新指标元信息

3. **更新macro_factor_config表**
   - 插入新指标计算配置
   - 如有特殊处理（如减基准值），在filter_params中注明

4. **如为衍生指标（如剪刀差）**
   - 在pipeline.py中添加数据加载逻辑
   - 参考 `_load_m1m2_scissor()` 实现

5. **重新计算因子**
   - 运行批量计算脚本

6. **验证**
   - 检查因子值合理性
   - 更新本文档
