## 一、项目目标

基于"宏观-中观-微观"三层决策框架，完成中观层（Phase 2.4）与微观层（Phase 2.5）的代码实现与回测验证。

**投资目标**: 卫星仓位（固定比例，如总资产10%）以行业ETF为标的，月频调仓，目标跟上市场beta、控制回撤，整体组合年化收益目标约8%。

**核心原则**:
- 中观负责"选池"，微观负责"择时与风控"
- 宏观层负责风险排除（极端象限清仓），非极端时期仅做降权
- 技术指标是"纪律工具"而非"预测工具"

### 2.3 ETF-行业映射规则（V5 TIE方法）

**核心指标**: TIE(Target Industry Exposure) = ETF成分股中属于该申万一级行业的权重之和

| 条件 | 映射关系 | 处理方式 |
|------|---------|---------|
| TIE ≥ 50% 且 (第一TIE - 第二TIE) ≥ 10% | **一级映射** | 纯度达标，可直接作为交易标的 |
| 30% ≤ TIE < 50% 或纯度差 < 10% | **备用映射** | 标注"⚠️高污染"，仅在无一级映射时备用 |
| TIE < 30% | **不建立映射** | 属宽基主题，剔除 |

**执行优先级**: 触发买入信号时，仅在一级映射ETF中选择近1月日均成交额最高者；若无一级映射，优先放弃该行业配置。

## 三、实施阶段

### 阶段1: Phase 2.4 中观行业比较框架（目标: 10个工作日）

#### Day 1-2: 模块1 — ETF映射引擎（TIE计算）
**目标**: 建立申万一级行业 ↔ ETF 的定量映射表

**输入**:
- etf_universe.csv（ETF列表）
- ETF成分股权重文件（复用Phase 2.2数据路径）
- index_daily（ETF价格数据，计算流动性）

**输出**:
- `industry_etf_mapping.csv`（行业→ETF映射表）
- `etf_tie_scores.csv`（各ETF的31个行业TIE得分）

**开发内容**:
1. 读取各ETF最新成分股权重文件
2. 按申万一级行业分类汇总权重（TIE计算）
3. 映射分级：一级/备用/不映射
4. 流动性筛选：按近1月日均成交额排序
5. 为每个行业确定首选ETF（一级映射中成交额最高者）

**验收标准**: 每个申万一级行业至少有1只一级映射或备用映射ETF

---

#### Day 3-4: 模块2+3 — 绝对趋势过滤 + 多周期动量
**目标**: 筛选上升趋势行业，计算相对强度得分

**模块2: 绝对趋势预筛选**
- 计算31个申万行业指数的MA60
- 每月末筛选：收盘价 > MA60 的行业进入后续
- 边界处理：上市不足60日的行业用可用数据计算

**模块3: 多周期动量与加速度**
- 相对价格序列: RP_t = 行业指数_t / 中证全指_t
- 中期动量(40%): Z(6个月超额收益)
- 长期动量(30%): Z(12个月超额收益)
- 动量加速度(30%): Z(Z(1M超额) - Z(3M超额))
- RS_score = 0.40 × 中期 + 0.30 × 长期 + 0.30 × 加速度

**验收标准**:
- 2015年1月至今每月末正确计算RS_score
- 高分组(Q1)平均超额收益显著高于低分组(Q5)

---

#### Day 5-6: 模块4+5 — 排名稳定性 + 优势池构建
**目标**: 构建"中观优势池"

**模块4: 排名稳定性**
1. 每月末按RS_score排名（1-N）
2. 回溯6个月排名，计算标准差 σ_rank
3. 缺失值惩罚：未通过MA60的月份排名 = N+1
4. Stability_score = Z(-σ_rank)

**模块5: 优势池构建**
- Composite_score = 0.60 × RS_score + 0.40 × Stability_score
- 准入规则（并集）:
  - 条件A: Composite_score > 0
  - 条件B: 排名前1/3 且 Composite_score > -0.5
- 输出: 每月"中观优势池"

**验收标准**:
- 2015-2024年每月优势池规模在3-15个行业
- 优势池行业下月平均超额收益 > 0

---

#### Day 7: 模块6 — 宏观协同（降权机制）
**目标**: 将宏观象限转化为对中观选股的风格约束

**实现内容**:
1. **极端象限清仓**: 失速衰退(P4) / 极端滞胀(P1) → 优势池清空
2. **非极端象限降权**:
   - 强势复苏/完美扩张/过热期: 防御型行业 × 0.8
   - 宽衰退/弱复苏: 强周期行业 × 0.8
   - 典型滞胀: 成长型行业 × 0.8
   - 震荡/观望/类衰退过渡: 不做降权
3. **行业周期敏感度标签**:
   - 高周期敏感: 有色、钢铁、煤炭、建材、机械、汽车、房地产
   - 中周期敏感: 化工、电力设备、电子、计算机、传媒、通信
   - 低周期敏感（防御）: 食品饮料、医药、公用事业、交运、银行、非银金融

**验收标准**: 不同宏观象限下，优势池构成与理论偏好一致

---

#### Day 8-9: 模块7 — 数据验证与单元测试
**验证一: RS_score预测能力**
- 分5组统计下个月平均超额收益
- 通过标准: Q1 > Q5 且 Q1显著为正

**验证二: MA60过滤有效性**
- MA60以上组 vs 以下组的下月收益对比
- 通过标准: 以下组显著低于以上组，且以下组均值为负

**验证三: 优势池质量**
- 统计每月优势池规模分布
- 验证优势池行业下月胜率 > 50%

### 阶段2: Phase 2.5 微观趋势验证（目标: 阶段1完成后，7-10天）

#### 模块8: 入场确认
**模式A（回调买入，优先）**:
- MA20 > MA60 > MA120（多头排列）
- MACD DIF > 0
- 收盘价距MA20在 ±3% 以内（强势回调）
- 或收盘价距MA60在 ±3% 以内，MACD绿柱缩短，过去5日累计跌幅 < 5%（深度回调）
- 过去5日累计跌幅 < 8%

**模式B（突破买入，补充）**:
- 收盘价首次上穿MA20不超过5个交易日
- MA20 > MA60 且 MA20已拐头向上
- MACD金叉或零轴附近

**说明**: 模式A优先于模式B

#### 模块9: 仓位分配
- 入选行业等权分配卫星总资金
- 单行业上限不超过卫星总资金的40%
- 最多持有5个行业，超过按得分取前5
- N=0时全部留现金或回归底仓

#### 模块10: 持仓期防守机制
**日频快速触发（无豁免）**:
- 单日跌幅 > 7% → 次日开盘平仓
- 连续3个交易日每日跌幅 > 2% 且累计 > 8% → 次日开盘平仓

**周频趋势结构监控（首周豁免）**:
- 趋势结构防线: MA20 < MA60 → 次日开盘平仓
- 动能防线: MACD DIF < DEA 且 DIF < 0 → 次日开盘平仓
- 拥挤防线: Crowding_score > 1.5 → 减仓50%
  - Crowding_score = 0.5 × Z(相对换手率) + 0.5 × Z(成交额占比)
  - 连续4周触发拥挤预警 → 直接平仓

#### 模块11: ETF执行
- 通过industry_etf_mapping.csv将行业转化为ETF
- 优先一级映射，无一级映射则放弃配置
- 同一行业多只一级映射ETF，选成交额最高者
- 次月第一个交易日以开盘价调仓

### 阶段3: 回测验证框架（目标: 阶段1+2完成后，7-10天）

#### 3.1 前提性验证
- **验证一**: RS_score预测能力（高分组Q1 vs 低分组Q5）
- **验证二**: MA60过滤有效性（以上组 vs 以下组）

#### 3.2 参数调优
| 参数 | 验证内容 |
|------|---------|
| 中期/长期/加速度权重（40/30/30） | 不同权重组合对下月超额收益的预测能力 |
| RS/Stability权重（60/40） | 稳定性得分的边际贡献 |
| 准入阈值（>0, 前1/3） | 不同阈值下优势池的规模与质量 |
| 入场容差（±3%） | 不同容差范围对入场胜率的影响 |
| 防线阈值（拥挤度1.5） | 不同阈值对回撤控制的贡献 |
| 卫星仓位比例（10%） | 不同比例下的整体组合风险收益特征 |

#### 3.3 完整回测
- 回测区间: 2015年1月 ~ 最新
- 调仓频率: 月频（月末计算，次月初执行）
- 监控频率: 日频（快速触发）+ 周频（趋势/动能/拥挤防线）
- 对比基准: 等权持有31个行业、沪深300、中证500
- 输出指标: 净值曲线、年化收益、最大回撤、夏普比率、胜率、Calmar比率

## 四、宏观-中观-微观协同流程

```
Step 1: 宏观守卫（月频）
  ├─ 查询最新宏观象限（macro_state_detail.csv）
  └─ 若为 失速衰退(P4) 或 极端滞胀(P1) → 清仓，终止

Step 2: 绝对趋势预筛选（月频）
  └─ 保留 收盘价 > MA60 的行业

Step 3: 中观筛选（月频）
  └─ 计算 Composite_score，构建"中观优势池"

Step 4: 宏观降权（月频）
  └─ 非偏好行业中观得分 × 0.8，重新排序

Step 5: 微观入场确认（月频）
  └─ 模式A优先，模式B补充，构建"最终候选池"

Step 6: 仓位分配（月频）
  └─ 等权分配，最多5个行业，单行业上限40%

Step 7: ETF映射与执行（月频）
  └─ 行业→ETF→次月初开盘价调仓

Step 8: 持仓期监控（日频+周频）
  ├─ 每日: 快速触发检查（无豁免）
  └─ 每周: 趋势/动能/拥挤防线（首周豁免）
```

## 七、注意事项

1. **版本管理**: 所有参数修改需记录变更日志，便于回滚和对比
2. **数据质量**: 每月调仓前需检查数据完整性（特别是1月IAV缺失的处理）
3. **边界情况**: 
   - 上市不足60日的新行业指数如何处理MA60
   - 行业指数停牌或数据缺失时的回退策略
   - ETF停牌或流动性枯竭时的替代方案
4. **回测诚实性**: 
   - 避免未来函数（如使用月末收盘价计算信号，次月开盘价执行）
   - 考虑交易滑点和冲击成本
   - 考虑ETF费率和跟踪误差
