# 中观行业比较与微观趋势验证方法论（Phase 2.4 & 2.5）
**版本**: v5.0 
**状态**: 方法论定稿，待工程落地与回测验证 
**更新说明**: 修复ETF映射逻辑失效、中观动量共线性及微观风控盲区，引入动量加速度与TIE纯度控制。

## 一、框架总览与设计哲学
### 1.1 三层决策分工
本方法论文档覆盖“宏观-中观-微观”三层决策框架中的中观层（Phase 2.4）与微观层（Phase 2.5）。

|层级|核心问题|输入数据|输出|当前状态|
|---|---|---|---|---|
|**宏观层** (Phase 2.3)|“经济处于什么阶段？哪些行业特征应受益？”|PMI、IAV、CPI、PPI、M2、社融等|宏观象限 + 风格倾向|✅ 已完成|
|**中观层** (Phase 2.4)|“哪些行业自身在走强且趋势在加速？”|申万行业指数价格|**中观优势池**|📋 本文档|
|**微观层** (Phase 2.5)|“趋势是否健康、是否可入场？持仓期间如何防守？”|优势池行业价格、成交数据|**最终候选池** + 仓位分配 + 防守规则|📋 本文档|

### 1.2 投资目标与框架定位

**投资目标**：底仓（风险平价稳健组合）占绝大部分资产，卫星仓位占固定小比例（如10%），以行业ETF为标的，月频调仓。卫星策略的目标不是获取超额收益，而是**跟上市场beta、控制回撤、从市场趋势中分一小杯羹**。整体组合年化收益目标约8%。

**框架定位**：
本框架中的技术指标（均线、MACD、拥挤度）不是“预测工具”，而是**“纪律工具”**：
- 均线排列：确保只在上升趋势中持有，不在下跌趋势中死扛
- MACD：确认趋势方向，在趋势动能衰竭时发出离场信号
- 拥挤度：在持仓过热时减仓，避免踩踏式暴跌
这些指标不追求精准抄底逃顶，只追求“该拿的时候拿着，该走的时候走掉”。滞后性和假信号在追求超额的框架中是缺陷，在本框架中是可接受的——少赚一点没关系，只要不亏大钱。

**核心设计原则**：
- **中观负责“选池”，微观负责“择时与风控”**：中观筛选出相对强势的行业，微观在中观池内确认入场时机、管理仓位和持仓期的安全防线。中观层不涉及任何交易决策。
- **纯价格驱动起步，保留基本面扩展接口**：当前中观信号完全基于价格数据，未来可叠加景气度验证层。
- **宏观层负责风险排除，而非收益放大**：宏观象限决定是否清仓，正常时期仅对非偏好行业做降权处理。
- **固定资金池管理卫星仓位**：卫星仓位占用总资产固定比例，与底仓资金严格隔离，盈亏互不传导。

### 1.3 未来扩展方向
本框架的优化方向不在A股行业轮动本身的精细打磨，而在于**增加不同收益来源的资产**（如海外股票、商品、债券等），通过多资产、多策略的分散化来优化整体组合。行业轮动卫星策略是整体组合中的一块“砖”，未来将有更多“砖”加入。


## 二、数据基础设施
### 2.1 行业指数
采用**申万一级行业指数**（2021版，31个）作为行业比较的标准标的。
- **数据源**：申万指数官网下载中心，或通过 Tushare `index_member_all` 接口获取日频价格。
- **基准指数**：**中证全指 (000985.SH)** 或 **国证A指 (399317.SZ)**，二者择一即可。两者均覆盖全部A股，日收益率与万得全A相关性 > 0.98。
- **不推荐沪深300**：沪深300仅覆盖大盘股，与31个申万一级行业存在系统性的大小盘风格偏差。
- 
### 2.2 ETF-行业专属映射表（V5修订）
由于最终交易标的为ETF，需建立申万行业与ETF之间的定量映射关系。为避免跨行业主题ETF造成“暴露污染”，废弃Jaccard成分股重叠度，改用**目标行业暴露度（TIE）**。
- **核心指标**：**目标行业暴露度（TIE, Target Industry Exposure）** = ETF成分股中属于该申万一级行业的**权重之和**。
- **主导归属原则**：一只ETF仅归属于其TIE最高的申万一级行业，避免一仆多主。
- **映射规则**：

|条件|映射关系|处理方式|
|---|---|---|
|归属行业TIE ≥ 50%，且 (第一TIE - 第二TIE) ≥ 10%|**一级映射**|纯度达标，可直接作为该行业的交易标的|
|归属行业 30% ≤ TIE < 50%，或纯度差 < 10%|**备用映射**|非纯行业暴露，标注“⚠️高污染”，仅在无一级映射时备用|
|归属行业 TIE < 30%|**不建立映射**|属宽基主题，剔除出行业轮动池|

- **执行优先级**：当某行业触发买入信号时，仅在其一级映射ETF中选择近1月日均成交额最高者执行；若无一级映射，优先放弃该行业配置（资金回归底仓），谨慎使用备用映射。


## 三、中观行业比较框架
**目标**：每月月末从31个申万一级行业中，筛选出自身处于上升趋势、趋势加速且相对强势稳定的行业，组成“中观优势池”。
**中观层职责**：在行业池中做横向比较，输出候选名单，不涉及任何交易决策。

### 3.1 绝对趋势预筛选（准入前置）
**经济含义**：排除自身处于下行趋势的行业，确保进入中观比较的行业都在“涨”，而非仅仅“跌得少”。
**规则**：
> 只有满足 `收盘价 > MA60`（60日移动平均线）的行业，才有资格进入后续的相对强度计算和优势池构建。

**说明**：
- MA60约对应一个季度的趋势。行业价格站上MA60，意味着中长周期趋势方向向上。
- 在普跌市，大量行业收盘价在MA60以下 → 通过绝对过滤的行业极少或为零 → 卫星仓位自然轻仓或空仓。
- 在结构性行情中，只有真正在涨的行业能通过此过滤。
- 在普涨市中，大量行业通过，相对比较负责从中选出最强的。

### 3.2 第一重：多周期动量与加速度（V5修订）
**经济含义**：捕捉通过绝对趋势过滤的行业，不仅在长周期有超额收益，且近期具有边际向上的动能加速，避免选入强弩之末的行业。

**计算步骤**：
1. **相对价格序列**： $RP_t = \frac{\text{行业指数价格}_t}{\text{基准指数价格}_t}$ 起始点归一化为1。
2. **三个子得分**：

| 子得分  | 计算方式                                   | 权重      | 捕捉内容                                     |
| ---- | -------------------------------------- | ------- | ---------------------------------------- |
| 中期动量 | 过去6个月行业超额收益（行业收益 − 基准收益），截面Z-score标准化  | **40%** | 中期趋势惯性（核心基石）                             |
| 长期动量 | 过去12个月行业超额收益，截面Z-score标准化              | **30%** | 长期基本面趋势（稳定器）                             |
| 动量加速 | `[Z(1个月超额收益) - Z(3个月超额收益)]` 的截面Z-score | **30%** | 截面排名的边际变化——排名提升（加速/反转启动）或排名下降（动能衰减/趋势末期） |

**相对强度得分**：
`RS_score = 0.40 × Z(6M超额) + 0.30 × Z(12M超额) + 0.30 × Z(1M动量 - 3M动量)`

**说明**：若1个月动量远强于3个月动量，说明趋势正在加速（右侧启动期）；反之则说明趋势衰竭，该因子给出负分，有效过滤高位滞涨行业。权重为起步设定，需通过回测验证后调优。


### 3.3 第二重：行业相对排名的稳定性（V5修订）

**经济含义**：行业内相对排名的稳定程度，反映其强势的可持续性。

**计算步骤**：
1. **截面排名**：每月末，将所有通过绝对趋势过滤的行业的 `RS_score` 从高到低排序，得到每个行业的排名（1–N）。
2. **排名标准差**：回溯过去**6个月**（含当月）的排名，计算每个行业排名的标准差 σ_rank。
3. **缺失值惩罚**：**若某行业在部分月份未通过绝对趋势过滤（无排名），该月排名统一赋值为 N+1（N为当月通过过滤的行业总数）**，计入标准差计算，避免短期跃升被误判为极度稳定。
4. **稳定性得分**： $Stability\_score = Z(-\sigma_{rank})$ 排名越稳定，得分越高。



### 3.4 中观综合得分与优势池构建

**综合得分**： $Composite\_score = 0.60 \times RS\_score + 0.40 \times Stability\_score$

**优势池规则（相对阈值准入）**：
行业进入“中观优势池”，需满足以下**任一**条件：
- **条件A（绝对门槛）**：`Composite_score > 0`——行业的综合得分（相对强度 + 稳定性）高于历史平均水平。
- **条件B（兜底机制）**：`Composite_score排名前1/3`，且 `Composite_score > -0.5`——即使绝对得分不高，但排名靠前且不极端弱势。
两者取并集。


## 四、微观趋势验证与交易控制框架

**目标**：在中观优势池内，对每个行业进行入场时机确认、仓位分配和持仓期安全监控。

**微观层职责**：回答“什么时候买、买多少、什么时候卖、什么时候止损”。所有交易层面的控制全部在微观层。

**技术指标的定位**：均线排列、MACD、拥挤度三者分工明确，作为持仓和风控的**纪律工具**，而非预测工具。

|指标|功能角色|核心问题|
|---|---|---|
|**均线排列**|趋势结构|“趋势是否健康、方向是否向上？”|
|**MACD**|动能监控|“趋势的推动力是在增强还是衰竭？”|
|**拥挤度**|热度监控|“是否已经过度拥挤，踩踏风险多大？”|

### 4.1 入场条件（月末计算，次月初执行）

中观优势池内的行业，满足以下**任一**入场模式，即可进入最终候选池：
**模式A：回调买入（优先，V5细化）**

|条件|要求|逻辑|
|---|---|---|
|趋势结构确认|MA20 > MA60 > MA120（均线多头排列）|上升趋势结构完好|
|动能确认|MACD DIF > 0（零轴上方运行）|中期动能向上|
|价格位置（强势回调）|收盘价距 MA20 在 ±3% 以内|回踩短期均线，提供安全边际|
|价格位置（深度回调）|收盘价距 MA60 在 ±3% 以内，**且 MACD柱状线为绿柱且开始缩短，过去5日累计跌幅 < 5%**|回踩中期均线极易变盘，必须附加动能衰竭确认，防接飞刀|
|跌幅限制|过去5日累计跌幅 < 8%|排除崩盘式下跌|

**模式B：突破买入（补充）**

|条件|要求|逻辑|
|---|---|---|
|刚突破|收盘价首次上穿 MA20 不超过5个交易日|捕捉启动点|
|趋势结构确认|MA20 > MA60 且 MA20 已拐头向上|短期趋势确立|
|动能确认|MACD金叉（DIF上穿DEA）或在零轴附近|动能配合突破|

**说明**：模式A优先于模式B——如果一个行业同时满足两种模式，按模式A（回调位）入场，成本更优。

**若优势池中没有任何行业满足入场条件**，则本月卫星仓位空仓，资金回归现金或底仓。

### 4.2 仓位分配规则

**总体原则**：卫星仓位采用固定资金池管理，占总资产固定比例（具体比例通过回测确定）。卫星仓位与底仓资金严格隔离，盈亏互不传导。

**分配规则**：
- 入选行业（最终候选池）之间等权分配卫星总资金。
- 若入选行业数量为 N（N ≤ 5），每个行业分配 `卫星总资金 / N`。
- **单行业仓位上限**：不超过卫星总资金的40%（N ≥ 3时自动满足；N=2时超出部分留现金；N=1时该行业仅占40%，其余60%留现金）。
- 若 N = 0，卫星总资金全部留现金或回归底仓。
**月内止损后的资金处理**：
- 如果月内防守机制触发导致某行业被平仓或减仓，释放的资金**不立即重新分配**，留作现金或回归底仓，等待下一个月末调仓时重新纳入分配。
**与底仓的关系**：
- 卫星仓位的盈亏不影响底仓的风险平价结构。
- 卫星仓位和底仓之间不做动态再平衡。

### 4.3 持仓期防守机制
#### 4.3.1 日频快速触发机制（极端事件监控，V5修订）
**目的**：捕获持仓期间出现的极端不利事件，无需等到周频检查。**极端止损是生存底线，不设任何豁免。**

**监控对象**：所有卫星持仓行业（**取消新仓豁免**）。

|条件|阈值|触发后处理|
|---|---|---|
|单日跌幅过大|当日跌幅 > 7%|次日开盘平仓|
|连续阴跌|连续3个交易日每日跌幅均 > 2%，且累计跌幅 > 8%|次日开盘平仓|
**数据需求**：仅需持仓行业的日频收盘价，计算量极小。

#### 4.3.2 周频趋势结构监控（V5修订）
**目的**：每周确认持仓行业的趋势结构是否仍然完好。趋势结构是稳定的状态描述，不因单日波动而改变，周频检查已足够。
**检查时间**：每周最后一个交易日收盘后执行。
**监控对象**：所有卫星持仓行业（仅保留首周新仓豁免）。

|防线|条件|触发后处理|含义|
|---|---|---|---|
|**趋势结构防线**|MA20 < MA60（短期均线下穿中期均线，死叉）|次日开盘平仓|上升趋势结构破坏|
|**动能防线**|MACD DIF < DEA（死叉）**且 DIF < 0**|次日开盘平仓|动能实质性衰竭，趋势反转预警|
|**拥挤防线**|Crowding_score > 1.5|次日开盘减仓50%，标记高拥挤预警|资金过热，踩踏风险上升|
**动能防线补充说明**：若MACD在零轴上方微弱死叉（DIF仍 > 0），可能是强势震荡，不立即平仓，但将日频快速触发阈值收紧为5%。

**拥挤防线的补充说明**：
- `Crowding_score = 0.5 × Z(相对换手率) + 0.5 × Z(成交额占比)`，其中相对换手率 = 过去5日日均换手率 / 过去60日日均换手率，成交额占比 = 行业过去5日日均成交额 / 全市场过去5日日均成交额。两者各自截面Z-score标准化后等权合成。
- 若触发拥挤防线，减仓50%，释放资金留现金或回归底仓。该行业标记为“⚠️高拥挤预警”。
- 下周检查时，若拥挤度仍高于1.5，继续维持半仓。若连续四周触发拥挤预警，第四周直接平仓。
- 若触发拥挤防线的同时，也触发趋势结构防线或动能防线，则直接平仓（不等减仓流程）。
- 
**豁免规则**：月初第1周（当月第1-5个交易日）新开仓的持仓，当周的周频检查豁免执行——给新仓位至少一周的观察期。


## 五、宏观-中观协同机制
**目标**：将宏观层的周期定位判断，转化为对中观选股的**风险排除**，而非收益放大。
### 5.1 极端象限清仓规则
若当月宏观象限为以下任一，则**卫星仓位清空，资金全部回归底仓**，越过所有中观/微观信号：
- **失速衰退**（P4）
- **极端滞胀**（P1）
### 5.2 非极端象限下的降权机制
|宏观象限|偏好行业特征|不匹配行业处理|
|---|---|---|
|强势复苏（P7）、完美扩张（P8）、过热期（P3）|高周期敏感（顺周期）|防御型行业中观得分 × 0.8|
|宽衰退（P5）、弱复苏（P6）|防御型（低周期敏感、高股息）|强周期行业中观得分 × 0.8|
|典型滞胀（P2）|上游资源、红利|成长型行业中观得分 × 0.8|
|震荡/观望（P10）、类衰退过渡（P9）|无显著偏好|不做任何降权处理|
- 0.8为起步乘数，通过回测可调优。
- 降权而非加权：匹配偏好的行业保持原得分，不匹配的行业得分打8折，让其自然排序后退。
**行业特征标签**：为31个申万一级行业预设“周期敏感度”标签（高/中/低），基于行业历史Beta或营收对GDP弹性确定。标签为静态预设，每半年复核一次。


## 六、最终卫星仓位生成流程
**调仓频率**：月频。每月最后一个交易日收盘后计算，次月第一个交易日执行。
**持仓监控频率**：
- 日频：快速触发机制（极端事件监控，无豁免）
- 周频：趋势结构、动能、拥挤防线检查（首周豁免）
**执行步骤**：
**Step 1 — 宏观守卫（月频）**：
- 查询最新宏观象限。
- 若为 `失速衰退` 或 `极端滞胀` → 卫星仓位清空，流程终止。
**Step 2 — 绝对趋势预筛选（月频）**：
- 31个申万一级行业中，仅保留 `收盘价 > MA60` 的行业进入后续流程。
**Step 3 — 中观筛选（月频）**：
- 对通过Step 2的行业，计算 `Composite_score`（含动量加速因子与排名缺失惩罚）。
- 按相对阈值准入规则（条件A或条件B），构建“中观优势池”。
**Step 4 — 宏观降权（月频）**：
- 根据当前宏观象限的风格倾向，对优势池内不匹配的行业中观得分 × 0.8。
- 降权后重新排序。
**Step 5 — 微观入场确认（月频）**：
- 对优势池内行业逐一进行入场条件验证（模式A优先，模式B补充）。
- 满足入场条件的行业进入“最终候选池”。
**Step 6 — 仓位分配（月频）**：
- 最终持有行业数量上限为 **5个**。若候选池超过5个，按综合得分降序取前5个。
- 入选行业等权分配卫星总资金，单行业上限不超过卫星总资金的40%。
- 若候选池不足5个，仅配置通过的那些，剩余资金留现金或回归底仓。
- 若候选池为0，卫星总资金全部留现金或回归底仓。
**Step 7 — ETF映射与执行（月频）**：
- 通过 ETF-行业专属映射表，将每个入选行业转化为对应的ETF（优先一级映射，无一级映射则放弃配置）。
- 若一个行业有多只一级映射ETF，选择近1个月日均成交额最高的那只。
- 次月第一个交易日以开盘价完成调仓。
**Step 8 — 持仓期监控（日频 + 周频）**：
- 每日：检查快速触发条件（任何时段新仓均不豁免），触发则次日开盘平仓。
- 每周最后一个交易日：检查趋势结构防线、动能防线、拥挤防线（首周新仓豁免），触发则次日开盘执行对应处置。

## 八、迭代与扩展路径
本框架的优化方向不在A股行业轮动本身的精细打磨，而在于多资产、多策略的分散化。行业轮动卫星策略是整体组合中的一块“砖”。
### 8.1 短期迭代（前提性验证 + 参数调优）
#### 8.1.1 前提性验证
在进行任何参数优化之前，需先验证框架的两个底层假设是否成立。这些验证本身就是参数选择的前置性研究——验证通过后，参数才正式生效。
**验证一：RS_score 的预测能力**
- **验证对象**：RS_score（**40/30/30加权**）的截面排名对下个月行业超额收益的预测能力。
- **做法**：取过去3-5年历史数据，每月月末计算各行业的RS_score，按得分高低分成5组，统计各组下个月的平均超额收益。
- **通过标准**：高分组（Q1）的平均超额收益持续高于低分组（Q5），且最高分组的超额收益在统计上显著为正。
- **若不通过**：缩短动量观察窗口（如将6M/12M改为3M/6M），或调整权重（如提高中期动量权重至70%），重新验证。
**验证二：MA60绝对趋势过滤的有效性**
- **验证对象**：收盘价 > MA60这一条件是否能过滤掉未来下跌概率大的行业。
- **做法**：每月月末将行业分为两组（MA60以上/以下），统计两组下个月的平均绝对收益和超额收益。
- **通过标准**：MA60以下组的绝对收益和超额收益均显著低于MA60以上组，且MA60以下组的绝对收益均值为负。
- **若不通过**：测试MA120或MA20作为替代过滤条件，选择过滤效果最优的均线。
#### 8.1.2 参数调优
前提性验证通过后，对以下参数进行回测调优：

|参数|验证内容|
|---|---|
|中观：中期/长期/加速度权重（40/30/30）|不同权重组合对下月超额收益的预测能力|
|中观：RS_score / Stability 权重（60/40）|稳定性得分的边际贡献|
|中观：准入阈值（Composite_score > 0，前1/3）|不同阈值下优势池的规模与质量|
|微观：入场模式（回调/突破）的参数|容差范围、跌幅限制、确认窗口|
|微观：卫星仓位比例与单行业上限|不同风险预算下的回撤与收益特征|
|微观：防线参数|拥挤度阈值、周频vs月频检查的对比|
### 8.2 中期扩展（行业轮动框架内）
- **中观层：引入景气度验证层**（分层验证，不混合加权）：当未来可获取部分行业的基本面景气度数据时，构建独立的景气度模块，对优势池内行业做验证——景气度扩张的行业保留，景气度收缩的行业标记背离并降权或排除。
- **微观层：引入RSRS（阻力支撑相对强度）**作为领先验证插件，辅助确认入场信号。
- **微观层：引入赫斯特指数**作为策略开关——高赫斯特指数时趋势策略更有效，低赫斯特指数时降低仓位或切换为反转策略。
### 8.3 长期扩展（多资产、多策略）
本框架的未来方向不在A股行业轮动本身的过度优化，而在于**增加不同收益来源**：
- **海外股票**：将中观行业比较框架复制到海外市场（如美股行业ETF），获取不同经济周期的beta收益
- **商品**：引入商品期货趋势跟踪策略，作为卫星仓位的另一块“砖”
- **债券**：在宏观象限指向宽衰退时，可配置长端利率债ETF
- **多策略配比**：底仓与卫星策略之间的比例可阶段性调整（一旦调整即为固定比例，不做频繁再平衡）
每次迭代仅在相应模块内进行，不破坏三层框架的整体结构。

_本方法论自2026年5月起实施。所有参数均为基于逻辑推理的起步设定，需通过历史回测验证其有效性与稳健性，并根据实证结果持续调优。_
