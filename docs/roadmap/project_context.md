# Project Context - Investment Agent

**最后更新**: 2026-04-23
**版本**: v1.0
**状态**: 稳定

---

## 1. 项目核心目标

### Phase 1: 真实投资组合跟踪（已完成）
- 读取持仓，自动获取行情，计算每日盈亏、权重与个股贡献
- 生成自然语言投资日报，并自动归档历史快照
- 支持股票、ETF、公募基金的混合持仓分析

### Phase 2: ETF 组合构建与研究（当前重点）
- 通过自然语言对话，为用户提供可重复的 ETF 组合研究报告
- 涵盖 ETF 筛选、组合回测、相关性/协方差分析、组合构建方法对比
- Agent 负责计算与呈现，人做最终投资决策，不直接产出交易信号

### 未来扩展
- 时序复盘：基于历史快照生成周度/月度趋势分析
- 模拟组合：虚拟账户与影子组合对比
- 交互看板：可选的 Streamlit 可视化界面

## 2. 技术栈

| 层级 | 技术组件 | 用途 |
| :--- | :--- | :--- |
| 开发辅助 | OpenCode / Cursor | 云端大模型辅助编程 |
| 大模型 | Qwen-Max / Plus (API) | 开发与运行的推理引擎 |
| 核心框架 | LangGraph | Skills编排与对话状态管理 |
| 数据存储 | SQLite + SQLAlchemy | 本地轻量级数据持久化 |
| 数值计算 | Pandas + Numpy | 时序数据处理 |
| 数据契约 | Pydantic (Lite) | 数据结构定义与接口约束 |
| 数据源 | iFinD / AkShare | iFinD为主，AkShare降级备用 |
| 技能架构 | Modular Skills | 独立封装的业务技能模块 |

---

## 3. 设计原则

1. **业务价值优先**：架构服务于功能，不为了架构而架构
2. **云端智能驱动**：全面拥抱云端大模型处理复杂金融逻辑
3. **结构化契约**：Pydantic定义清晰的数据接口
4. **快速迭代交付**：先MVP，再逐步完善
5. **方法论先行**：复杂金融逻辑先形成文档，确认后实施
