# Phase 2.1 Summary - 核心池构建

**阶段**: Phase 2.1
**名称**: 核心池构建
**时间**: 2026-04-20 至 2026-04-22
**状态**: 已完成

---

## 1. 完成内容

### 1.1 核心池策略构建
- 基于风险平价（Risk Parity）方法构建核心底仓
- 对比三个组合方案：组合A（多元化）、组合B（低波动）、组合C（A∪B合并）
- **组合B胜出**: 夏普比率1.12，年化波动4.65%，最大回撤-8.10%

### 1.2 回测引擎开发
- 实现通用回测引擎，支持指数替代、费率扣除、现金资产分离
- 节假日数据补齐（2019-2026年224天）
- 全NaN日期过滤，避免节假日空仓导致权重清零

### 1.3 数据基础设施
- 创建`index_universe.csv`指数映射表（10只指数）
- 创建`index_daily`表，导入6,551条全收益指数记录
- 费率配置：从etf_universe_v2.csv获取各ETF费率（0.2%-0.8%）

### 1.4 关键修复
- 数据存储规则：上市前数据丢弃，上市后0值转NULL
- 协方差估计：滚动60交易日
- 再平衡频率：月度

---

## 2. 关键假设与决策

### 2.1 策略选择
- **风险平价 vs 等权重**: 选择风险平价，因为核心池需要稳健穿越周期
- **目标波动率**: 接受核心池低波动特性（1-2%），不强行缩放至6%
- **现金处理**: 短融ETF作为现金替代，不参与风险平价求解

### 2.2 数据假设
- **指数替代**: ETF上市前使用全收益指数数据，按比例缩放，扣除ETF费率
- **费率扣除**: 仅扣除指数替代期间的ETF费率（年化total_fee/365）
- **数据边界**: iFinD试用账号限制（2021-04-19起），早期数据使用Wind手工补齐

### 2.3 组合设计
- **组合B（低波动）作为底仓**: 10只ETF，以红利策略为核心
- **核心80% + 卫星20%**: 固定仓位分配，卫星策略用于增强
- **资产数量**: 10-13只为最优区间，19只合并后反而降低效率

---

## 3. 当前缺陷

### 3.1 数据局限
- **历史数据短**: 最早仅2019年，无法回测更早历史（如2015年股灾）
- **跨境资产**: 纳指、恒生等数据依赖单一数据源
- **费率简化**: 未完全模拟实际交易摩擦成本

### 3.2 模型简化
- **再平衡成本**: 未考虑冲击成本、滑点
- **静态权重**: 核心池权重固定，未根据市场环境动态调整
- **单一策略**: 仅使用风险平价，未对比其他组合构建方法

### 3.3 资产冗余
- **组合B内部相关性**: 多只红利ETF可能高度相关
- **现金比例**: 当前现金资产权重为0，未强制保留5-10%现金

---

## 4. 后续扩展点

### 4.1 资产精简
- **相关性过滤**: 剔除与现有资产相关性>0.9的ETF
- **边际贡献评估**: 计算每只资产加入后的夏普比率边际提升
- **PCA分析**: 识别真正独立的收益来源

### 4.2 动态调整
- **策略切换**: 基于宏观指标在组合A和B之间动态切换权重
- **风险预算分配**: 调整A和B的风险预算而非切换底层资产
- **卫星池整合**: 将A中的高弹性资产作为卫星策略

### 4.3 多策略对比
- **等权重回测**: 对比风险平价与等权重表现
- **目标波动率**: 测试带目标波动率缩放的风险平价
- **最小方差**: 对比最小方差组合
