# 流动性维度方法论（V8 Final）

## 一、核心设计理念

流动性维度的目标是判断货币与信用的松紧状态及其边际变化方向。与增长、通胀维度保持完全同构：**水平衡量绝对位置，方向衡量边际动量**。

- **水平判定**：由数量型指标（M2、社融）决定，反映中长期信用条件的松紧情况。
- **方向判定**：由价格型指标（央行政策利率OMO及其与市场利率DR007的传导关系）决定，**同时接纳数量指标的方向信号作为辅助验证**，捕捉流动性的边际变化与政策意图。

**原价格否决权逻辑废弃**：不再因DR007短期高于政策利率而降级流动性状态，因其混淆了“央行主动引导”与“市场被动收紧”，且在政策利率上调周期会导致系统性误判。新框架将价格信号回归其应有角色——**方向的领先指标**。

---

## 二、数据源

- **M2**：广义货币供应量存量同比增速。月度，中国人民银行。
- **社融**：社会融资规模存量同比增速。月度，中国人民银行。
- **DR007**：存款类机构7天质押式回购利率月度加权均值。日频合成月度值，CFETS。
- **OMO利率**：央行7天期逆回购操作利率。月度取月末有效利率；日频用于拐点识别。

## 三、水平判定（沿用现有逻辑）

对M2和社融的当月同比序列分别做单向HP滤波（λ=129600），提取周期项。水平判定规则：

| M2周期项 | 社融周期项 | 流动性水平 |
|:---|:---|:---|
| ≥ 0（货币扩张） | ≥ 0（信用扩张） | **双宽** |
| ≥ 0（货币扩张） | < 0（信用收缩） | **宽货币紧信用** |
| < 0（货币收缩） | ≥ 0（信用扩张） | **紧货币宽信用** |
| < 0（货币收缩） | < 0（信用收缩） | **双紧** |

水平判定部分保持稳定，不涉及价格信号。

## 四、方向判定

方向判定分为三步：先识别央行意图作为主信号，再通过传导验证进行修正，最后与数量方向合成。

### 4.1 央行意图识别（初始方向）

**定义**：依据过去6个月内OMO利率的累计净变化方向判定央行是边际收紧还是边际宽松。

- **净变化** = 当期OMO利率 − 6个月前OMO利率（若期间存在多次调整，则按调整幅度累加）
- **阈值**：累计上调 > 10bp → **收紧**，初始方向 ↓  
  　　　　累计下调 > 10bp → **宽松**，初始方向 ↑  
  　　　　变化在±10bp以内 → **中性**，初始方向 →

**规则说明**：
- 6个月窗口对应货币政策向实体经济传导的典型时滞。
- 10bp阈值过滤技术性微调（OMO最小调整步长为5bp或10bp）。
- 6个月内若出现先加息后降息，净变化以最终结果为准；若净变化虽小但12个月内曾有显著加息/降息且当前利率处于高位/低位，维持此前的方向判定（以防加息暂停期被误判为中性）。

### 4.2 传导验证（方向修正）

央行意图能否落地，取决于市场利率（DR007）是否跟随政策利率变动，且两者趋势的**先后关系**决定了是主动引导还是被动追赶。

#### 4.2.1 趋势拐点识别方法

**预处理**：
- OMO：取日频政策利率。DR007：取日频加权利率。
- 分别计算20交易日的简单移动平均（SMA20），以滤除日间噪声。

**突破识别**：
- 计算SMA20的5日差分（diff = SMA20(t) − SMA20(t−5)）。
- 趋势形成：diff连续5个交易日同号（正或负），方向由负转正视为上行突破，由正转负视为下行突破。
- 拐点日期：确认为趋势形成的第一天（即连续同号的第一天）。
- **OMO突变跳过机制**：若OMO单日变动 ≥ 5bp，直接判定为趋势拐点，以变动日为拐点日，不等待SMA确认。

**拐点列表**：对OMO和DR007分别生成历史突破点的日期及方向（上行/下行）。

#### 4.2.2 先后关系判定

取最近一次**同向突破点**进行比较。如两者方向相反，直接判定传导受阻（方向→）。

方向相同时，比较拐点日期：
- 若 OMO拐点日 < DR007拐点日 − 20个交易日（约1个月）：**央行主动引导**
- 若 DR007拐点日 < OMO拐点日 − 20个交易日：**央行被动追赶**
- 否则：视为几乎同步（有效调控）

**若某一方在近12个月内无任何突破点**（趋势水平无方向），则默认另一方为引领者；双方均无趋势则方向→。

#### 4.2.3 方向修正表

| 初始方向（央行意图） | 先后关系 | 修正后价格方向 |
|:---|:---|:---|
| 收紧（↓） | OMO主动引导 | ↓ |
| 收紧（↓） | DR007被动追赶/同期/传导受阻 | → |
| 宽松（↑） | OMO主动引导 | ↑ |
| 宽松（↑） | DR007被动追赶/同期/传导受阻 | → |
| 中性（→） | 任何 | → |

修正后的价格方向记为 `price_dir`，取值 ↑ / ↓ / →。

### 4.3 数量方向提取（沿用V7逻辑）

M2和社融各自经统一流水线（HP滤波→Z-score→自适应偏离→防抖状态机）产生 `trend_dir`，两者合成规则同增长维度：
- 同向取同向
- 一非中性一中性取非中性
- 一上一下冲突 → 中性

最终得到数量方向 `qty_dir`。

### 4.4 价格方向与数量方向的冲突仲裁

价格方向（短端信号）是央行意图的领先指标，数量方向（长端结果）是滞后验证。两者为上下游关系，非并列。冲突时采用**价格主导、数量验证**的仲裁规则：

| 价格方向 `price_dir` | 数量方向 `qty_dir` | 最终方向 | 标记 |
|:---|:---|:---|:---|
| ↑ | ↑ | ↑ | — |
| ↑ | → | ↑ | — |
| ↑ | ↓ | ↑ | ⚠️数量价格背离 |
| ↓ | ↑ | ↓ | ⚠️数量价格背离 |
| ↓ | ↓ | ↓ | — |
| ↓ | → | ↓ | — |
| → | ↑ | ↑ | ⚠️短端未确认 |
| → | ↓ | ↓ | ⚠️短端未确认 |
| → | → | → | — |

规则要点：
1. 价格方向明确时（↑/↓），最终方向服从价格方向，数量方向仅作为附加验证信息。
2. 当价格方向为中性（→）时，说明央行无边际意图，此时以数量方向的惯性为准，但打上“短端未确认”标记，以降低方向信号的置信度。
3. 价格方向与数量方向相反时，属于深度背离，保留价格方向，并打上“数量价格背离”标记——提示策略层短端与长端在拉锯。

## 五、流动性状态合成

组合水平与方向，形成最终标签，示例：

- `双宽↑`：货币信用双扩张，且边际上仍在加速宽松
- `紧货币宽信用↑`（标记：⚠️短端未确认）：信用在扩张，数量方向显示向上，但央行无明确意图，方向可信度下降
- `双紧↓`：货币信用双收缩，且央行主动收紧
- `宽货币紧信用↑`（标记：⚠️数量价格背离）：央行在宽松，但数量信号仍在收缩，可能存在传导时滞或结构性矛盾

流动性状态标签与增长、通胀维度状态标签一同进入宏观象限映射引擎。

## 六、与原价格否决权的过渡说明

- **废除**：原“DR007高于政策利率超20bp即降级流动性状态”的硬性规则。
- **替代**：价格信号转化为方向判定的有机组成部分，解决政策利率上调周期中的误判问题，同时保留对传导受阻状态的识别能力。
- **兼容**：数量水平判定未变，历史回测中与流动性相关的变化仅体现在方向标签上，象限映射结果将更为稳健。

## 七、工程实施伪代码示例

```python
def liquidity_direction(omo_daily, dr007_daily, m2_monthly, sfs_monthly, current_date):
    # 1. 央行意图初始方向
    omo_6m_ago = omo_daily.loc[current_date - pd.DateOffset(months=6)]
    net_change = omo_daily.loc[current_date] - omo_6m_ago
    if net_change > 0.10:
        intent = 'tighten'
    elif net_change < -0.10:
        intent = 'ease'
    else:
        intent = 'neutral'

    # 2. 传导验证 → 价格方向
    omo_bp = find_breakpoints(omo_daily, current_date)
    dr007_bp = find_breakpoints(dr007_daily, current_date)
    lead = compare_lead(omo_bp, dr007_bp)
    if intent == 'tighten' and lead == 'omo_lead':
        price_dir = '↓'
    elif intent == 'ease' and lead == 'omo_lead':
        price_dir = '↑'
    else:
        price_dir = '→'

    # 3. 数量方向（沿用V7逻辑）
    qty_dir = quantity_direction(m2_monthly, sfs_monthly, current_date)

    # 4. 冲突仲裁
    if price_dir == '↑':
        final_dir = '↑'
        flag = '⚠️数量价格背离' if qty_dir == '↓' else None
    elif price_dir == '↓':
        final_dir = '↓'
        flag = '⚠️数量价格背离' if qty_dir == '↑' else None
    else:  # price_dir == '→'
        final_dir = qty_dir
        flag = '⚠️短端未确认' if qty_dir != '→' else None

    return final_dir, flag
```
